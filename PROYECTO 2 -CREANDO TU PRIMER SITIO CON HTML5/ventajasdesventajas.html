lo bueno y lo malo del lenguaje PHP

Ventajas del lenguaje PHP

  • Lenguaje totalmente libre y abierto.
  • Posee una curva de aprendizaje muy baja.
  • Los entornos de desarrollo son de rápida y fácil configuración.
  • Fácil de instalar
  • Fácil acceso e integración con las bases de datos.
  • Posee una comunidad muy grande.
  • Es el lenguaje con mayor usabilidad en el mundo.
  • Es un lenguaje multiplataforma.
  • Completamente orientado al desarrollo de aplicaciones web dinámicas y/o páginas web con acceso a una Base de Datos.
  • El código escrito en PHP es invisible al navegador ya que se ejecuta al lado del servidor y los resultados en el navegador es HTML.
  • Posee una versatilidad para la conexión con la mayoría de base de datos que existen en la actualidad.

desventajas

  • PHP es un lenguaje de programación del lado del servidor diseñado para varios propósitos. Pero se utiliza principalmente para crear contenido dinámico en un sitio web.
  • Es muy popular debido a su naturaleza de código abierto y características flexibles. Es bastante sencillo para principiantes, pero los programadores profesionales también pueden usarlo para funciones más complejas.
  • Aprender y comprender PHP definitivamente puede mejorar sus habilidades de desarrollador: puede crear cualquier tipo de sitio web, convertirse en un desarrollador experimentado de WordPress o incluso crear una aplicación web algún día.
  • Debe aprender a usar marcos PHP: estos marcos permiten a los programadores agregar funcionalidad sin escribir código adicional. Sin embargo, este es un trabajo adicional al aprender la utilidad del marco.
  • La calidad del marco es variable: aunque la mayoría de los entornos de trabajo son de código abierto y gratuitos, la calidad variará según el entorno en el que se utilice. Es decir, algunos entornos laborales tienen poco apoyo y son muy lentos.
  • Puede ralentizar la velocidad de las aplicaciones web: muchos bancos de trabajo nos proporcionan potentes funciones y herramientas para acelerar los sitios web complejos. Por supuesto, para aplicaciones simples, no es necesario tener características tan avanzadas. Por lo tanto, en algunos casos, esto puede tener un impacto negativo en el rendimiento de la Web.
